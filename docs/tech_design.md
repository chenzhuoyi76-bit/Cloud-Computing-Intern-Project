# 技术方案设计

整个系统分为三个部分：

### 1 用户交互层

- 制作一个简易的 VS Code 插件（Javascript/Typescript）本地加载
- 负责接收用户自然语言输入，将请求发给后端API
- 展示系统结果

### 2 后端服务层

- 使用 Flask 搭建后端 API （本地运行）
- 后端调用大模型API（计划使用OpenAI）做用户意图识别
- 根据识别结果做任务分发（python）
- 执行指定仓库的测试、部署、部署后的状态监测等任务（初步选择为拉取指定python仓库，建立新工作目录，执行测试，部署到docker，监测该docker）
- 记录日志和结果

### 3 数据存储层

- 计划暂时使用SQLite（便于快速开发和验证）
- 用于存储请求记录、任务执行日志,存储执行结果和报告


### 4 后续扩展
- 用户交互层后期考虑搭建Web UI 并部署到服务器
- 数据库后期考虑MySQL
